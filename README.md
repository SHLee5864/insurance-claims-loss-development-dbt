# 🏦 Insurance Claims Loss Development — dbt Project

The system **Loss Development Triangle / LDF / Ultimate Loss** with Insurance claims data
Pipeline for the calculation by dbt + DuckDB(dev) / Databricks(prod).

---

## 📐 Architecture

RAW (seed/CSV) → STG (view) → INT (table) → MART (table)

| Layer | Role |
|-------|------|
| RAW   | Stock the data from CSV (Generated by Python) |
| STG   | Casting the type of the variables, rename, surrogate key |
| INT   | Refactoring AY·DevMonth·Valuation |
| MART  | Calculation of Triangle, LDF, Ultimate Loss, Loss Ratio |

---

## 🗂 Model Lineage

stg_claims + stg_policy + stg_calendar
  → int_claim_base
    → int_payments_enriched → int_claim_monthly_paid → int_dev_month
    → int_reserve_snapshot_enriched

int_dev_month + int_reserve_snapshot_enriched
  → mart_paid_triangle
  → mart_incurred_triangle → mart_ldf → mart_ultimate_loss
                                          → mart_loss_ratio_by_ay

int_exposure_summary → mart_exposure_alignment

## ⚙️ Quick Start (DuckDB / Dev)

# 1. Enviroment
python -m venv venv && venv\Scripts\activate  # Windows
pip install dbt-duckdb dbt-utils

# 2. 패키지 설치
dbt deps

# 3. seed 적재 (RAW CSV)
dbt seed

# 4. 전체 모델 실행
dbt run

# 5. 테스트
dbt test

## 📊 Model of MART 

| 모델 | 설명 |
|------|------|
| mart_paid_triangle      | AY × DevMonth accumulated amount of paid Triangle |
| mart_incurred_triangle  | Paid + Reserve Incurred Triangle |
| mart_ldf                | Volume-Weighted LDF + accumulated LDF |
| mart_ultimate_loss      | Ultimate Loss by AY |
| mart_loss_ratio_by_ay   | Loss Ratio by AY |
| mart_exposure_alignment | Alignement of AY × Region Exposure |

---

## 🔢 Calculation of LDF

Volume-Weighted Average LDF:
  ldf = SUM(incurred_to) / SUM(incurred_from)  ( by all AYs)
Cumulative LDF = EXP(SUM(LOG(ldf))) OVER (ORDER BY dev_month_to)
